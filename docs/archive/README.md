# 文档归档说明

本目录包含已过时或被替代的历史文档，仅供参考。

## 📂 归档结构
- **v1.0/** - v1.0版本的临时文档和报告
- **bug-fixes-history/** - 已被统一标准替代的bug修复文档
- **temporary/** - 开发过程中的临时草稿

## ⚠️ 重要提示
归档文档中的信息可能已过时，请优先查看当前文档：
- **风险等级标准** → docs/standards/risk-level-standard.md
- **项目状态** → /CONTEXT.md
- **版本历史** → /VERSION.md
- **文档索引** → docs/INDEX.md

## 🔍 查找历史信息
如需查阅历史修复记录或版本演进，可在此目录中搜索。

## 📋 归档规则

### 何时归档文档
1. **文档被新版本替代** → 归档到对应版本目录
2. **临时草稿完成正式化** → 归档到temporary/
3. **Bug修复后建立标准** → 归档到bug-fixes-history/
4. **版本淘汰（如v0.x）** → 归档到版本目录

### 归档命名规范
- 保持原始文件名，或添加版本后缀（如 `-v1.md`）
- 在原文件位置添加指向新文档的引用
- 在归档目录添加README说明
