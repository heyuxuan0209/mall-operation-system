## 📝 编写说明

### 快照编写原则
1. **静态冻结**: 版本发布后不再修改
2. **精简自包含**: 200-400行讲清楚一个版本
3. **增量对比**: 突出"相比上一版本的变更"
4. **只列要点**: 不展开细节，细节见详细文档

### 目标行数
- 最少: 200行（太少说不清楚）
- 最多: 400行（太多失去"快照"意义）
- 理想: 250-350行

- [ ] 行数在200-400之间？
