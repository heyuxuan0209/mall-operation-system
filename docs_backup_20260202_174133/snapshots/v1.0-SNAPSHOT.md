## 业务规则摘要

> **注意**：以下为v1.0权重标准（已废弃）。v2.0及以后请查看最新标准文档：`docs/standards/risk-level-standard.md`

### 健康度计算（5维度）- v1.0标准 ❌已废弃
- **租金缴纳** (25%)：逾期天数、欠费金额
- **经营表现** (25%)：营业额、坪效、客流量
- **现场品质** (20%)：陈列、环境、服务 - ⚠️ v2.0已调整为30%
- **顾客满意度** (15%)：评分、投诉 - ⚠️ v2.0已调整为10%
- **抗风险能力** (15%)：现金流、合同期限 - ⚠️ v2.0已调整为10%

### 风险等级标准（4等级）
| 等级 | 分数 | 颜色 |
|------|------|------|
| 高风险 | 0-59 | 红色 |
| 中风险 | 60-79 | 橙色 |
| 低风险 | 80-89 | 黄色 |
| 无风险 | 90-100 | 绿色 |

### 任务状态机
```
pending → in_progress → evaluation → completed
                ↓
            cancelled
```

### 案例匹配算法
```
匹配分数 = 业态匹配(40%) + 标签匹配(60%) + 症状加分
```

## 已知问题和限制

### 技术债务
1. **LocalStorage限制** - 仅本地存储，无法跨设备
2. **无认证授权** - 无用户系统
3. **无移动端优化** - 响应式设计不完善
4. **数据容量限制** - LocalStorage 5-10MB限制

### 功能缺失
1. **无团队协作** - 单用户模式
2. **无消息通知** - 无任务提醒
3. **无多商场管理** - 只支持单商场
4. **无现场巡店** - 缺少移动端工具
